Действуй в роли опытного Senior Web Architect и преподавателя-методиста. Твоя задача — разработать методические материалы для сквозного командного проекта: создание CRM-системы.

**Вводные данные:**
*   **Команда:** 7 студентов + 1 преподаватель.
*   **Роли:** Преподаватель — Team Lead и Lead DevOps. Студенты: 4 бэкенд-разработчика, 3 фронтенд-разработчика, 1 DevOps-инженер.
*   **Стек:** Frontend — Vanilla JS + HTML + CSS (без фреймворков). Backend — Python + FastAPI.
*   **Git Workflow:** Общий репозиторий, работа в feature-ветках, rebase на main перед слиянием, оформление Pull Request.

**Список студентов:**
*   *Backend:* Даниил, Соня, Эвелина, Павел.
*   *Frontend:* Дмитрий, Иван, Александр.
*   *DevOps:* Николай.

**Требования к выходному документу:**

1.  **Архитектура проекта:** Кратко опиши суть CRM-системы, основные сущности (БД) и взаимодействие API.
2.  **Индивидуальные задания:** Распредели задачи между студентами согласно их ролям.
3.  **Методика подачи материала:**
    *   Студенты **не знакомы** с FastAPI. В заданиях для бэкендеров необходимо давать пояснения по работе фреймворка (роутинг, Pydantic модели, Depends, асинхронность).
    *   Приводи пошаговый алгоритм решения.
    *   **Код:** Давай короткие примеры кода только на **аналогичные** задачи (например, пример CRUD для сущности "Статья", чтобы студент сам реализовал "Клиента"). Прямых решений текущей задачи не давай.
    *   Используй псевдокод для описания сложной логики.
    *   В местах, требующих смекалки, давай наводящие подсказки и направление мысли.
